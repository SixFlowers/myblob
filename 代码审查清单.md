# 代码审查清单

> 每次编写或修改代码详解文档时，必须对照此清单逐一检查

## 一、必须对照源代码验证的关键点

### 1.1 类型定义检查（最重要！）

| 检查项 | AnyBlob 真实定义 | 常见错误 |
|--------|----------------|---------|
| **HttpRequest** | `enum class Method : uint8_t { GET, PUT, POST, DELETE }` | ❌ 用 `std::string method` |
| **HttpRequest.headers** | `std::map<std::string, std::string> headers` | ❌ 用 `std::vector<std::pair<...>>` |
| **HttpResponse.code** | `enum class Code : uint8_t { OK_200, CREATED_201, ... }` | ❌ 用 `int status_code` |
| **HttpResponse.body** | **没有 body 字段！** 通过 `DataVector<uint8_t>` 传递数据 | ❌ 用 `std::string body` |
| **CloudService** | `enum class CloudService : uint8_t` **不是类！** | ❌ 用 `class CloudService` |

### 1.2 类和接口检查

| 文件 | 必须验证的内容 |
|------|---------------|
| `connection.hpp` | 类名、方法签名、返回值类型 |
| `connection_manager.hpp` | **返回值类型**：AnyBlob 返回 `int32_t fd`，不是 `shared_ptr<Connection>` |
| `http_client.hpp` | **没有 HttpClient 类**！AnyBlob 使用 `HTTPProvider` 和 `TaskedSendReceiver` |
| `provider.hpp` | `CloudService` 必须是枚举，方法签名必须一致 |
| `data_vector.hpp` | 必须正确引用，DataVector 是模板类 |

### 1.3 命名空间检查

| 检查项 | 正确 | 错误 |
|--------|------|------|
| AnyBlob 命名空间 | `anyblob::` | `myblob::` (除非明确说明是简化版) |
| 文档中的示例 | 应保持 `myblob::` 命名空间，但需注明这是简化版 |

## 五、1. 文件引用检查（重要！）

> ⚠️ **最容易犯的错误**：重复定义已经存在的类型！

在编写代码详解之前，必须先检查目标项目中是否已经存在相关定义：

### 必须检查的文件

| 文件 | 作用 | 检查要点 |
|------|------|---------|
| `cloud_service.hpp` | CloudService 枚举 | **不要重复定义 CloudService** |
| `http_request.hpp` | HttpRequest 结构 | AnyBlob 使用枚举+map |
| `http_response.hpp` | HttpResponse 结构 | AnyBlob 使用枚举+map |
| `data_vector.hpp` | 数据容器 | 必须正确引用 |
| `provider.hpp` | Provider 基类 | 方法签名必须一致 |

### 检查清单

- [ ] 检查目标项目是否有 `cloud_service.hpp`
- [ ] 如果有，引用它，不要重复定义 `CloudService`
- [ ] 检查 `HttpRequest`/`HttpResponse` 是否已有定义
- [ ] 如果有，使用已有的，不要创建新的
- [ ] 检查命名空间是否正确（`anyblob::` vs `myblob::`）

### 错误示例

```cpp
// ❌ 错误：在 provider.hpp 中重复定义 CloudService
#pragma once
#include "some_other_file.hpp"  // 已经包含了 CloudService

namespace myblob::cloud {
    // ❌ 这里不需要定义 CloudService，已经在 cloud_service.hpp 中了
    enum class CloudService : uint8_t { ... };  // 重复！
    
    class Provider {
        CloudService type_;  // 直接使用
    };
}
```

### 正确示例

```cpp
// ✅ 正确：引用已有的 cloud_service.hpp
#pragma once
#include "cloud_service.hpp"  // 引用已有的

namespace myblob::cloud {
    // ✅ 不需要定义 CloudService，直接使用
    class Provider {
        CloudService type_;  // 直接使用 cloud_service.hpp 中的类型
    };
}
```
