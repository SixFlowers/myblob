#### 架构变化

```cpp
// ❌ 版本 4：单请求处理
void sendRequest() {
    connect();
    send();
    recv();
    disconnect();
}

// ✅ 版本 5：批量多路复用
void Transaction::execute() {
    // 1. 建立所有连接
    for (auto& msg : messages_) {
        int fd = connMgr_->connect(...);
        pollSocket.send({fd, EventType::write});
    }
    
    // 2. 批量等待
    while (!allDone) {
        pollSocket.submit();  // poll() 等待
        
        while (auto* req = pollSocket.complete()) {
            processRequest(req);
        }
    }
}
```

#### poll 多路复用原理

```
┌─────────────────────────────────────────────────────┐
│              PollSocket 批量处理流程                  │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. 添加请求到 poll 监控                              │
│     pollfds = [fd1, fd2, fd3, ...]                  │
│                                                     │
│  2. poll() 等待（单线程同时监控多个 socket）           │
│     └── 返回：哪些 socket 可读/可写                   │
│                                                     │
│  3. complete() 获取已就绪的请求                       │
│     └── 处理每个完成的请求                           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### TLSConnection 核心设计

```cpp
class TLSConnection {
private:
    SSL* ssl_;                    // OpenSSL SSL 句柄
    BIO* internalBio_;            // 内部 BIO
    BIO* networkBio_;             // 网络 BIO
    
public:
    // 使用 BIO pair：internalBio ↔ networkBio
    // 应用程序读写 internalBio
    // 网络数据读写 networkBio
    
    bool send(const uint8_t* data, size_t len, size_t& sent);
    bool recv(uint8_t* data, size_t len, size_t& received);
};
```

### 📋 版本 9：AWS 签名版

**目标**：实现 AWS S3 签名，对标 AnyBlob 的 AWS/AWSSigner

#### AWS V4 签名流程

```cpp
class AWSSigner {
    std::string signRequest(
        const std::string& method,
        const std::string& url,
        const std::string& date,
        const std::string& accessKey,
        const std::string& secretKey,
        const std::string& region,
        const std::string& service
    );
};
```

#### 签名步骤

1. **规范请求**：构建规范化的请求字符串
2. **签名字符串**：`"AWS4-HMAC-SHA256\n" + date + "\n" + scope + "\n" + hash(canonical)`
3. **签名密钥**：从 secret key 派生：`kDate → kRegion → kService → kSigning`
4. **签名**：`HMAC-SHA256(signingKey, stringToSign)`
