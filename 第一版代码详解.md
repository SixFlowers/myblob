## 项目介绍

### 什么是 MyBlob？

**MyBlob** 是一个简单的云对象存储下载管理器。它的工作原理就像一个"万能下载器"，可以从不同的云服务（如 AWS S3、Azure、GCP、MinIO 等）下载文件。

**类比理解**：想象你要从不同的快递公司（顺丰、圆通、京东）取包裹，MyBlob 就是那个可以统一处理所有快递公司包裹的"万能取件员"。

### 主要功能

1. **下载文件**：从 HTTP/HTTPS/MinIO 服务器下载文件
2. **支持多种协议**：HTTP、HTTPS、S3、Azure、GCP、MinIO
3. **范围下载**：支持只下载文件的一部分（断点续传的基础）
4. **基础加密**：支持 HTTPS 加密传输

## 计算机网络基础知识

### 什么是 HTTP？

**HTTP（HyperText Transfer Protocol）** 是互联网上应用最广泛的网络协议，用于传输网页、图片、视频等数据。

**类比**：HTTP 就像邮寄包裹的"标准信封格式"，规定了如何填写收件人、发件人、内容等信息。

### HTTP 请求和响应

当你访问一个网页时，流程如下：

```
用户浏览器 <--HTTP请求--> Web服务器
用户浏览器 <--HTTP响应--> Web服务器
```

#### HTTP 请求的组成部分

```
GET /index.html HTTP/1.1          <-- 请求行（方法 路径 协议版本）
Host: www.example.com             <-- 请求头（服务器地址）
Connection: close                 <-- 请求头（连接类型）
                                   
                                   <-- 空行（分隔请求头和请求体）
[可选的请求体内容]                  <-- 请求体（如POST提交的数据）
```

#### HTTP 响应的组成部分

```
HTTP/1.1 200 OK                   <-- 状态行（协议版本 状态码 状态信息）
Content-Type: text/html            <-- 响应头（数据类型）
Content-Length: 1234              <-- 响应头（数据长度）

                                   <-- 空行
<html>...</html>                  <-- 响应体（实际的数据内容）
```

### HTTP 方法

| 方法         | 含义   | 使用场景      |
| ---------- | ---- | --------- |
| **GET**    | 获取资源 | 下载文件、获取网页 |
| **POST**   | 提交数据 | 上传文件、提交表单 |
| **PUT**    | 上传资源 | 更新/创建文件   |
| **DELETE** | 删除资源 | 删除文件      |

### HTTP 状态码

| 状态码     | 含义    | 示例         |
| ------- | ----- | ---------- |
| **200** | 成功    | GET 成功返回数据 |
| **206** | 部分成功  | Range 请求成功 |
| **400** | 请求错误  | 错误的请求格式    |
| **401** | 未授权   | 需要登录       |
| **403** | 禁止访问  | 没有权限       |
| **404** | 未找到   | 文件不存在      |
| **500** | 服务器错误 | 服务器崩溃      |

### HTTPS 和 SSL/TLS

**HTTPS = HTTP + SSL/TLS**

**SSL/TLS** 是一种加密协议，可以在传输过程中保护数据安全。

**类比**：

- **HTTP**：寄信时用透明信封，别人可以看到内容
- **HTTPS**：寄信时用加密信封，只有收件人能打开

### 什么是 Socket？

**Socket（套接字）** 是网络通信的端点，就像电话的听筒。

**类比**：

- Socket = 电话听筒
- IP 地址 = 电话号码
- 端口号 = 分机号

### TCP 连接过程

```
客户端                              服务器
   |                                 |
   |  1. 发起连接请求 (SYN)          |
   |------------------------------> |
   |                                 |
   |  2. 确认连接 (SYN+ACK)          |
   |<------------------------------ |
   |                                 |
   |  3. 确认收到 (ACK)              |
   |------------------------------> |
   |                                 |
   |  双向通信开始                   |
   |<==============================>|
   |                                 |
```

### 什么是端口号？

一台电脑（IP 地址）上运行着很多程序，端口号用来区分不同的程序。

| 端口号      | 常用服务         |
| -------- | ------------ |
| **80**   | HTTP（未加密的网页） |
| **443**  | HTTPS（加密的网页） |
| **21**   | FTP（文件传输）    |
| **22**   | SSH（远程登录）    |
| **3306** | MySQL（数据库）   |
| **9000** | MinIO（对象存储）  |

### DNS 是什么？

**DNS（Domain Name System）** 将域名（如 `www.example.com`）转换为 IP 地址（如 `93.184.216.34`）。

**类比**：DNS 就像电话本，把人名（域名）转换成电话号码（IP 地址）。

### 什么是 HTTP Range 请求？

Range 请求允许只下载文件的一部分。

**应用场景**：

1. **断点续传**：下载中断后，从上次中断的地方继续
2. **并行下载**：把文件分成多部分，同时下载加速
3. **视频播放**：在线播放视频时，先加载开头部分

**示例**：

```
Range: bytes=0-1023       <-- 下载前 1024 字节
Range: bytes=1024-2047    <-- 下载第 1024-2047 字节
Range: bytes=1024-        <-- 从第 1024 字节下载到结尾
```

## 云存储基础知识

### 什么是云存储？

**云存储** 是通过网络远程存储数据的服务，数据存储在云服务商的服务器上。

**类比**：

- **本地硬盘**：把钱存在自己家里的保险箱
- **云存储**：把钱存到银行的保险箱（更安全、更方便）

### 什么是对象存储？

**对象存储（Object Storage）** 是一种存储数据的方式，把每个文件当作一个"对象"来管理。

**特点**：

- **无限扩展**：可以存储海量数据
- **成本低廉**：按实际使用量付费
- **高可靠性**：数据自动备份

**类比**：对象存储就像邮局的"智能快递柜"，每个文件是一个快递，可以存储在无数个柜子中。

### 主流云存储服务商

| 服务商           | 产品名称             | 特点            |
| ------------- | ---------------- | ------------- |
| **Amazon**    | AWS S3           | 市场领导者，功能最全    |
| **Microsoft** | Azure Blob       | 企业级应用强        |
| **Google**    | GCP Storage      | 与 Google 云集成好 |
| **Oracle**    | OCI Object       | 性价比高          |
| **IBM**       | IBM Cloud Object | 混合云强          |
| **MinIO**     | MinIO            | 开源、可私有部署      |

### 云存储的 URL 格式

#### 标准 HTTP/HTTPS

```
http://server.com/bucket/file.txt
https://server.com/bucket/file.txt
```

#### AWS S3

```
s3://bucket-name/region/file.txt
```

#### MinIO（私有部署的对象存储）

```
minio://server:port/bucket:region/file.txt
```

### 云存储的核心概念

#### 1. Bucket（存储桶）

- **概念**：类似"文件夹"，用来存放文件
- **类比**：就像一个仓库的仓位

#### 2. Object（对象）

- **概念**：存储的文件，包括数据和元数据
- **类比**：仓库里的每个包裹

#### 3. Region（区域）

- **概念**：云服务器的物理位置
- **类比**：仓库所在的城市

#### 4. Endpoint（端点）

- **概念**：访问云存储的网络地址
- **类比**：仓库的收货地址

### 认证机制

大多数云存储需要身份认证：

#### 1. 访问密钥（Access Key）

- **Access Key ID**：用户名
- **Secret Access Key**：密码

#### 2. 签名机制

云服务商使用复杂的签名算法验证请求的合法性。

**类比**：就像在快递单上签名，证明是你本人寄的。

### MinIO 简介

**MinIO** 是一个开源的对象存储服务器，可以在自己的电脑上运行。

**特点**：

- ✅ 轻量级，安装简单
- ✅ 兼容 S3 API（可以使用 S3 的工具访问）
- ✅ 适合开发和测试
- ✅ 免费开源

## 代码详解

### 1. 数据容器 (data\_vector.hpp)

**文件路径**：`/home/root1/myblob/include/utils/data_vector.hpp`

**功能**：实现一个简单的动态数组容器，类似于标准库的 `std::vector`。

**为什么需要它**？

- 标准库的 `std::vector` 功能太多，消耗更多资源
- 我们的需求很简单：存储字节数据
- 可以自己控制内存分配

```cpp
#pragma once  // 防止头文件被重复包含
#include <cassert>           // 断言头文件
#include <cstdint>          // 整数类型定义
#include <cstring>          // 内存操作函数
#include <memory>           // 智能指针

namespace myblob::utils {   // 命名空间，避免名字冲突

// ============================================================
// 模板类 DataVector<T>
// ============================================================
// 这是一个通用的动态数组容器
// 模板参数 T 表示元素的类型
// 例如：DataVector<uint8_t> 存储字节数据
// ============================================================

template <typename T>
class DataVector {
private:
    // ============================================================
    // 私有成员变量
    // ============================================================
    
    /// _capacity: 数组的容量（最多能存多少元素）
    /// 注意：容量可能比实际大小（_size）大，这样可以在不重新分配内存的情况下添加元素
    uint64_t _capacity;
    
    /// _size: 当前数组中实际存储的元素数量
    uint64_t _size;
    
    /// _dataOwned: 智能指针，拥有数据的所有权
    /// unique_ptr 是独占所有权的智能指针
    /// 当 unique_ptr 被销毁时，它指向的内存也会被自动释放
    /// 这样做的好处是：不用担心内存泄漏
    std::unique_ptr<T[]> _dataOwned;
    
    /// _data: 指向数据的原始指针
    /// 这里分开存储有两个原因：
    /// 1. 支持拥有数据（_dataOwned 不为空）
    /// 2. 支持引用外部数据（_dataOwned 为空，直接指向外部数据）
    T* _data;

public:
    // ============================================================
    // 构造函数
    // ============================================================

    /// 默认构造函数
    /// 创建一个空的数组，没有任何数据
    /// 初始状态：容量=0，大小=0，数据指针=nullptr
    constexpr DataVector() : _capacity(0), _size(0), _data(nullptr) {}

    /// 带初始容量的构造函数
    /// 预先分配一定大小的内存，避免后续添加元素时频繁重新分配
    /// 使用 explicit 防止隐式类型转换
    /// 参数 cap: 初始容量
    explicit constexpr DataVector(uint64_t cap) : _capacity(0), _size(0), _data(nullptr) {
        resize(cap);  // 调用 resize 设置容量
    }

    /// 从指针范围构造
    /// 从 [start, end) 区间复制数据
    /// 参数 start: 起始指针
    /// 参数 end: 结束指针（不包含）
    /// 例如：DataVector<uint8_t>(ptr, ptr + 10) 复制 10 个元素
    constexpr DataVector(const T* start, const T* end) : _capacity(0), _size(0), _data(nullptr) {
        assert(end - start >= 0);  // 确保范围有效
        resize(static_cast<uint64_t>(end - start));  // 调整大小
        std::memcpy(data(), start, size() * sizeof(T));  // 复制数据
    }

    /// 引用外部数据的构造函数（不拥有所有权）
    /// 用于操作已存在的内存区域，而不是分配新内存
    /// 参数 ptr: 指向外部数据的指针
    /// 参数 capacity: 数据的容量（以 T 为单位，不是字节）
    /// 例如：DataVector<uint8_t>(buffer, 1024) 引用 1024 字节的缓冲区
    constexpr DataVector(T* ptr, uint64_t capacity) : _capacity(capacity), _size(0) {
        _data = ptr;  // 直接使用外部指针，不拥有所有权
    }

    // ============================================================
    // 公共成员函数
    // ============================================================

    /// 获取数据指针
    /// 返回指向数组数据的指针
    [[nodiscard]] constexpr T* data() { return _data; }

    /// 获取常量数据指针（用于只读访问）
    /// const 版本，防止修改数据
    [[nodiscard]] constexpr const T* cdata() const { return _data; }

    /// 获取当前数组大小（元素个数）
    [[nodiscard]] constexpr uint64_t size() const { return _size; }

    /// 获取数组容量（最多能存多少元素）
    [[nodiscard]] constexpr uint64_t capacity() const { return _capacity; }

    /// 清空数组（但保留内存）
    /// 注意：只是把大小设为 0，数据仍然在内存中
    /// 如果要释放内存，需要调用 shrink_to_fit() 或重新构造
    constexpr void clear() { _size = 0; }

    /// 检查是否拥有数据所有权
    /// 返回 true 表示拥有所有权（可以重新分配内存）
    /// 返回 false 表示引用外部数据（容量固定）
    [[nodiscard]] constexpr bool owned() { return _dataOwned || !_capacity; }

    /// 预留内存容量
    /// 如果新容量大于当前容量，分配更多内存
    /// 如果新容量小于等于当前容量，什么都不做
    /// 参数 cap: 新的容量大小（元素个数）
    /// 注意：不会改变数组大小（_size），只改变容量（_capacity）
    constexpr void reserve(uint64_t cap) {
        if (_capacity < cap) {  // 只在需要更大容量时操作
            // 检查是否拥有所有权
            if (!_dataOwned && _capacity)
                throw std::runtime_error("Pointer not owned, thus size is fixed!");
            
            // 分配新的内存
            auto swap = std::unique_ptr<T[]>(new T[cap]());
            
            // 如果有现有数据，复制到新内存
            if (_data && _size)
                std::memcpy(swap.get(), data(), _size * sizeof(T));
            
            // 交换指针
            _dataOwned.swap(swap);  // 新的 unique_ptr 替换旧的
            _data = _dataOwned.get();  // 更新数据指针
            
            // 更新容量
            _capacity = cap;
        }
    }

    /// 调整数组大小
    /// 如果新大小大于当前容量，会分配更多内存
    /// 如果新大小小于当前容量，保留现有内存
    /// 参数 size: 新的元素个数
    constexpr void resize(uint64_t size) {
        if (size > _capacity) {  // 需要更多内存
            reserve(size);  // 先分配内存
        }
        _size = size;  // 更新大小
    }

    /// 在数组末尾添加一个元素
    /// 如果容量不足，会自动扩容（容量翻倍）
    /// 参数 value: 要添加的元素值
    constexpr void push_back(const T& value) {
        if (_size >= _capacity) {  // 容量不足
            reserve(_capacity == 0 ? 1 : _capacity * 2);  // 扩容为 2 倍（或初始为 1）
        }
        _data[_size++] = value;  // 添加元素并增加大小
    }

    /// 下标运算符重载
    /// 允许使用 [] 访问元素
    /// 参数 index: 元素索引
    /// 返回: 元素的引用
    constexpr T& operator[](uint64_t index) { return _data[index]; }
    constexpr const T& operator[](uint64_t index) const { return _data[index]; }

    /// 转移数据所有权
    /// 返回智能指针并清空 DataVector
    /// 调用者获得数据的所有权，需要负责释放内存
    /// 返回: unique_ptr 智能指针
    [[nodiscard]] constexpr std::unique_ptr<T[]> transferBuffer() {
        return std::move(_dataOwned);  // std::move 转移所有权
    }
};

} // namespace myblob::utils
```

### 2. HTTP 客户端头文件 (http\_client.hpp)

**文件路径**：`/home/root1/myblob/include/network/http_client.hpp`

**功能**：定义 HTTP 客户端的接口，用于发送 HTTP 请求和接收响应。

```cpp
#pragma once  // 防止头文件被重复包含
#include "../utils/data_vector.hpp"  // 引入数据容器
#include <atomic>         // 原子操作，用于多线程
#include <condition_variable>  // 条件变量，用于线程同步
#include <cstdint>        // 整数类型
#include <memory>         // 智能指针
#include <mutex>          // 互斥锁
#include <string>         // 字符串
#include <thread>         // 线程
#include <vector>         // 动态数组

namespace myblob::network {  // 网络相关命名空间

// ============================================================
// HTTP 响应结构体
// ============================================================
struct HttpResponse {
    /// status_code: HTTP 状态码
    /// 200 = 成功, 404 = 未找到, 500 = 服务器错误 等
    int status_code;
    
    /// body: 响应的数据体（字节数组）
    /// 例如：下载的文件内容、API 返回的 JSON 数据
    utils::DataVector<uint8_t> body;
    
    /// error_message: 错误信息（如果有的话）
    /// 当 success 为 false 时，这个字段包含错误原因
    std::string error_message;
    
    /// success: 请求是否成功
    /// true = 状态码在 200-299 范围内
    /// false = 其他情况
    bool success;

    /// 默认构造函数
    HttpResponse() : status_code(0), success(false) {}
};

// ============================================================
// HTTP 请求结构体
// ============================================================
struct HttpRequest {
    /// method: HTTP 方法
    /// 常见值：GET, POST, PUT, DELETE
    std::string method;
    
    /// host: 目标服务器地址
    /// 例如：www.example.com
    std::string host;
    
    /// path: 请求路径
    /// 例如：/index.html, /api/data
    std::string path;
    
    /// body: 请求体（用于 POST、PUT 等方法）
    std::string body;
    
    /// headers: HTTP 头信息
    /// 键值对列表，例如：
    ///   "Content-Type" -> "application/json"
    ///   "Authorization" -> "Bearer xxx"
    std::vector<std::pair<std::string, std::string>> headers;
    
    HttpRequest() {}  // 默认构造函数
    
    /// 构造函数
    /// 参数 m: 方法, h: 主机, p: 路径
    HttpRequest(const std::string& m, const std::string& h, const std::string& p) 
        : method(m), host(h), path(p) {}
};

// ============================================================
// HTTP 客户端类
// ============================================================
class HttpClient {
public:
    /// 构造函数
    /// 初始化 OpenSSL（用于 HTTPS 加密）
    HttpClient();
    
    /// 析构函数
    /// 清理 OpenSSL 资源
    ~HttpClient();

    // ============================================================
    // GET 请求方法
    // ============================================================

    /// 从 URL 下载内容
    /// 参数 url: 完整的 URL，如 http://example.com/file.txt
    /// 返回: HTTP 响应
    HttpResponse get(const std::string& url);
    
    /// 发送 GET 请求（普通 HTTP）
    /// 参数 host: 服务器地址, path: 路径, port: 端口
    HttpResponse get(const std::string& host, const std::string& path, uint16_t port = 80);
    
    /// 发送 GET 请求（支持 HTTPS）
    /// 参数 host: 服务器地址, path: 路径, port: 端口, ca_file: 证书文件（可选）
    HttpResponse get(const std::string& host, const std::string& path, uint16_t port,
                     const std::string& ca_file);

    /// 下载文件（支持断点续传）
    /// 参数 url: 文件URL, offset: 起始字节, length: 要下载的字节数
    /// 例如：download(url, 1024, 4096) 下载第 1024-5119 字节
    HttpResponse download(const std::string& url, uint64_t offset = 0, uint64_t length = 0);

private:
    // ============================================================
    // 私有辅助方法
    // ============================================================

    /// 创建 TCP 连接到服务器
    /// 参数 host: 服务器地址（域名或IP）, port: 端口号
    /// 返回: socket 文件描述符（类似文件句柄）
    /// 失败返回 -1
    int create_socket(const std::string& host, uint16_t port);
    
    /// 发送 HTTP 请求
```

```cpp
/// 从 URL 中提取端口号
uint16_t HttpClient::extract_port(const std::string& url) {
    size_t start = url.find("://") + 3;
    size_t end = url.find("/", start);
    std::string host_port = url.substr(start, end - start);
    
    // 查找冒号
    size_t colon = host_port.find(":");
    if (colon != std::string::npos) {
        // 有端口号，转换为整数
        return static_cast<uint16_t>(std::stoi(host_port.substr(colon + 1)));
    }
    
    // 没有端口号，根据协议返回默认值
    return use_tls(url) ? 443 : 80;  // HTTPS 默认 443，HTTP 默认 80
}

// ============================================================
// create_socket - 创建网络连接
// ============================================================
// 功能：建立到服务器的 TCP 连接
// 参数：host 服务器地址（域名或IP）, port 端口号
// 返回：socket 文件描述符（成功），-1（失败）
// ============================================================
int HttpClient::create_socket(const std::string& host, uint16_t port) {
    // 1. 创建 socket
    // AF_INET: IPv4
    // SOCK_STREAM: TCP（面向连接）
    // 0: 自动选择协议（TCP）
    int sockfd = socket(AF_INET, SOCK_STREAM, 0);
    if (sockfd < 0) return -1;  // 创建失败

    // 2. 解析域名获取 IP 地址
    // gethostbyname 将域名转换为 IP 地址结构
    struct hostent* server = gethostbyname(host.c_str());
    if (server == nullptr) {
        close(sockfd);  // 清理资源
        return -1;  // 解析失败
    }

    // 3. 设置服务器地址结构
    struct sockaddr_in addr;
    std::memset(&addr, 0, sizeof(addr));  // 清零结构体
    addr.sin_family = AF_INET;  // IPv4
    // 复制 IP 地址到结构体
    std::memcpy(&addr.sin_addr.s_addr, server->h_addr, server->h_length);
    addr.sin_port = htons(port);  // 转换为网络字节序

    // 4. 建立连接
    if (connect(sockfd, (struct sockaddr*)&addr, sizeof(addr)) < 0) {
        close(sockfd);
        return -1;  // 连接失败
    }

    // 5. 优化：禁用 Nagle 算法
    // Nagle 算法会延迟发送小数据包，禁用它可以降低延迟
    int flag = 1;
    setsockopt(sockfd, IPPROTO_TCP, TCP_NODELAY, &flag, sizeof(flag));

    return sockfd;  // 返回 socket 描述符
}

// ============================================================
// send_request - 发送 HTTP 请求
// ============================================================
// 功能：将 HTTP 请求数据发送到服务器
// 参数：sockfd socket描述符, request HTTP请求
// 返回：服务器响应
// ============================================================
HttpResponse HttpClient::send_request(int sockfd, const HttpRequest& request) {
    // 1. 构建 HTTP 请求字符串
    std::string request_str = request.method + " " + request.path + " HTTP/1.1\r\n";
    
    // 2. 添加必要的头部
    request_str += "Host: " + request.host + "\r\n";  // 服务器地址
    request_str += "Connection: close\r\n";  // 告诉服务器处理完请求后关闭连接
    
    // 3. 添加自定义头部
    for (const auto& header : request.headers) {
        request_str += header.first + ": " + header.second + "\r\n";
    }
    
    // 4. 如果有请求体，添加 Content-Length
    if (!request.body.empty()) {
        request_str += "Content-Length: " + std::to_string(request.body.size()) + "\r\n";
    }
    
    // 5. 结束请求头
    request_str += "\r\n";
    
    // 6. 添加请求体（如果有）
    if (!request.body.empty()) {
        request_str += request.body;
    }

    // 7. 发送请求
    // send() 类似于 write()，但用于网络
    ssize_t sent = send(sockfd, request_str.c_str(), request_str.size(), 0);
    if (sent < 0) {
        // 发送失败
        HttpResponse resp;
        resp.success = false;
        resp.error_message = "Failed to send request";
        return resp;
    }

    // 8. 接收响应
    return recv_response(sockfd);
}

// ============================================================
// recv_response - 接收 HTTP 响应
// ============================================================
// 功能：从服务器接收并解析 HTTP 响应
// 参数：sockfd socket描述符
// 返回：解析后的响应对象
// ============================================================
HttpResponse HttpClient::recv_response(int sockfd) {
    HttpResponse response;
    utils::DataVector<uint8_t> buffer(8192);  // 8KB 接收缓冲区
    std::string header_buffer;  // 存储 HTTP 头部
    bool headers_complete = false;  // 头部是否接收完成
    size_t content_length = 0;  // Content-Length 值
    size_t body_received = 0;  // 已接收的 body 字节数

    // 循环接收数据
    while (true) {
        // 接收数据
        ssize_t n = recv(sockfd, 
                        buffer.data() + header_buffer.size(),  // 写入位置
                        buffer.capacity() - header_buffer.size() - 1,  // 可用空间
                        0);  // 默认选项
        
        // n <= 0 表示连接关闭或出错
        if (n <= 0) break;

        // 将新接收的数据追加到缓冲区
        header_buffer.append(buffer.data() + header_buffer.size(), n);

        // 如果还没接收完头部
        if (!headers_complete) {
            // 查找空行（"\r\n\r\n" 是 HTTP 头部的结束标志）
            size_t header_end = header_buffer.find("\r\n\r\n");
            if (header_end != std::string::npos) {
                // 提取头部部分
                std::string headers = header_buffer.substr(0, header_end);
                
                // 解析状态码
                size_t status_pos = headers.find("HTTP/");
                if (status_pos != std::string::npos) {
                    size_t status_end = headers.find("\r\n", status_pos);
                    std::string status_line = headers.substr(status_pos, status_end - status_pos);
                    
                    // 状态码在第 9-11 个字符
                    size_t code_pos = status_line.find(" ");
                    if (code_pos != std::string::npos) {
                        response.status_code = std::stoi(status_line.substr(code_pos + 1, 3));
                    }
                }

                // 解析 Content-Length
                size_t cl_pos = headers.find("Content-Length:");
                if (cl_pos != std::string::npos) {
                    size_t cl_end = headers.find("\r\n", cl_pos);
                    std::string cl_str = headers.substr(cl_pos + 15, cl_end - cl_pos - 15);
                    content_length = std::stoul(cl_str);
                }

                // 提取 body 部分（头部后面的数据）
                std::string remaining = header_buffer.substr(header_end + 4);
                header_buffer = remaining;  // 清空 header_buffer
                headers_complete = true;  // 标记头部接收完成

                // 初始化响应 body
                response.body.reserve(content_length > 0 ? content_length : 8192);
                
                // 将剩余数据转为字节并添加到 body
                for (char c : remaining) {
                    response.body.push_back(static_cast<uint8_t>(c));
                }
                body_received = remaining.size();

                // 如果已经收到完整的 body，可以提前退出
                if (content_length > 0 && body_received >= content_length) {
                    break;
                }
            }
        } else {
            // 头部已完成，将数据添加到 body
            for (ssize_t i = 0; i < n; i++) {
                response.body.push_back(static_cast<uint8_t>(header_buffer[i]));
            }
            body_received += n;
            
            // 如果已经收到完整的 body，退出循环
            if (content_length > 0 && body_received >= content_length) {
                break;
            }
        }
    }

    // 判断请求是否成功
    // HTTP 2xx 表示成功
    response.success = (response.status_code >= 200 && response.status_code < 300);
    
    // 如果失败，设置错误信息
    if (!response.success && response.error_message.empty()) {
        response.error_message = "HTTP request failed with status: " + std::to_string(response.status_code);
    }

    return response;
}
```

### 5. 云存储提供者实现 (provider.cpp)

**文件路径**：`/home/root1/myblob/src/cloud/provider.cpp`

**功能**：实现云存储提供者的功能，包括 URL 解析、请求创建等。

```cpp
#include "cloud/provider.hpp"
#include <algorithm>      // 算法库
#include <cstring>        // 字符串操作

namespace myblob::cloud {

// ============================================================
// 静态方法实现
// ============================================================

/// 检查 URL 是否为远程文件
/// 通过检查 URL 是否以已知的协议前缀开头
bool Provider::isRemoteFile(const std::string& url) {
    // 遍历所有已知的协议前缀
    for (size_t i = 0; i < remoteFileCount; i++) {
        // 如果 URL 以该前缀开头，返回 true
        if (url.find(remoteFile[i]) == 0) {
            return true;
        }
    }
    return false;
}

/// 根据 URL 获取云服务商类型
CloudService Provider::getCloudService(const std::string& url) {
    // 使用 if-else 判断协议前缀
    // 这种方式比循环更直观，也稍微高效一点
    if (url.find("https://") == 0) return CloudService::HTTPS;
    if (url.find("http://") == 0) return CloudService::HTTP;
    if (url.find("s3://") == 0) return CloudService::AWS;
    if (url.find("azure://") == 0) return CloudService::Azure;
    if (url.find("gs://") == 0) return CloudService::GCP;
    if (url.find("oci://") == 0) return CloudService::Oracle;
    if (url.find("ibm://") == 0) return CloudService::IBM;
    if (url.find("minio://") == 0) return CloudService::MinIO;
    
    // 默认返回 Local
    return CloudService::Local;
}

/// 创建云存储提供者工厂方法
/// 这是"工厂模式"的典型应用
std::unique_ptr<Provider> Provider::makeProvider(
    const std::string& url,
    bool https,
    const std::string& access_key,
    const std::string& secret_key,
    void* send_receiver_handle
) {
    // 1. 确定云服务商类型
    CloudService service = getCloudService(url);
    
    // 2. 根据类型创建对应的提供者
    // 这里只实现了 HTTP 和 MinIO，其他类型需要扩展
    std::string addr;
    uint16_t port;

    // HTTP/HTTPS 协议处理
    if (service == CloudService::HTTP || service == CloudService::HTTPS) {
        // 提取协议前缀
        std::string protocol = https ? "https://" : "http://";
        std::string remaining = url.substr(protocol.size());
        
        // 找到路径开始的位置
        size_t slash_pos = remaining.find('/');
        
        // 提取 "主机:端口" 部分
        std::string host_port = (slash_pos == std::string::npos) 
            ? remaining 
            : remaining.substr(0, slash_pos);
        
        // 解析主机和端口
        size_t colon_pos = host_port.find(':');
        if (colon_pos != std::string::npos) {
            // 格式：host:port
            addr = host_port.substr(0, colon_pos);
            port = static_cast<uint16_t>(std::stoi(host_port.substr(colon_pos + 1)));
        } else {
            // 格式：host
            addr = host_port;
            port = https ? 443 : 80;  // 根据协议选择默认端口
        }
        
        // 创建 HTTP 提供者
        return std::make_unique<HTTPProvider>(addr, port, https);
    }

    // MinIO 协议处理
    if (service == CloudService::MinIO) {
        // MinIO URL 格式：minio://host:port/bucket:region/file
        // 跳过 "minio://" 前缀
        std::string remaining = url.substr(7);
        
        // 找到路径开始的位置
        size_t slash_pos = remaining.find('/');
        
        // 提取 "主机:端口" 部分
        std::string host_port = (slash_pos == std::string::npos) 
            ? remaining 
            : remaining.substr(0, slash_pos);
        
        // 解析主机和端口
        size_t colon_pos = host_port.find(':');
        if (colon_pos != std::string::npos) {
            addr = host_port.substr(0, colon_pos);
            port = static_cast<uint16_t>(std::stoi(host_port.substr(colon_pos + 1)));
        } else {
            // MinIO 默认端口是 9000
            addr = host_port;
            port = 9000;
        }
        
        return std::make_unique<HTTPProvider>(addr, port, false);
    }

    // 不支持的类型，返回空指针
    return nullptr;
}

/// 下载文件
network::HttpResponse Provider::download(const std::string& file_path, uint64_t offset, uint64_t length) {
    // 创建 HTTP 客户端
    network::HttpClient client;
    
    // 构建完整的 URL
    std::string full_url;
    if (type_ == CloudService::HTTP || type_ == CloudService::HTTPS) {
        // HTTP/HTTPS: 使用指定的地址和端口
        full_url = (https_ ? "https://" : "http://") + address_ + ":" + std::to_string(port_) + file_path;
    } else {
        // 其他类型：直接使用 file_path
        full_url = file_path;
    }

    // 调用客户端下载
    return client.download(full_url, offset, length);
}

// ============================================================
// HTTPProvider 实现
// ============================================================

/// 构造函数
HTTPProvider::HTTPProvider(const std::string& addr, uint16_t port, bool https)
    : address_(addr), port_(port), https_(https) {
    // 设置类型
    type_ = https ? CloudService::HTTPS : CloudService::HTTP;
}

/// 创建 GET 请求
std::unique_ptr<utils::DataVector<uint8_t>> HTTPProvider::createGetRequest(
    const std::string& file_path,
    uint64_t offset,
    uint64_t length
) {
    // 创建请求数据容器
    auto request = std::make_unique<utils::DataVector<uint8_t>>();
    
    // 构建 HTTP 请求字符串
    std::string request_str = "GET " + file_path + " HTTP/1.1\r\n";
    request_str += "Host: " + address_ + "\r\n";
    request_str += "Connection: close\r\n";
    
    // 如果有范围请求，添加 Range 头
    if (offset > 0 || length > 0) {
        std::string range = "bytes=" + std::to_string(offset) + "-";
        if (length > 0) {
            range += std::to_string(offset + length - 1);
        }
        request_str += "Range: " + range + "\r\n";
    }
    
    // 结束头部
    request_str += "\r\n";
    
    // 调整容器大小
    request->resize(request_str.size());
    
    // 复制数据
    std::memcpy(request->data(), request_str.c_str(), request_str.size());
    
    return request;
}
```

### 6. 示例程序 (simple\_download.cpp)

**文件路径**：`/home/root1/myblob/example/simple_download.cpp`

**功能**：演示如何使用 MyBlob 下载文件。

```cpp
#include "cloud/provider.hpp"  // 引入云存储提供者
#include <fstream>             // 文件操作
#include <iostream>            // 输入输出
#include <string>              // 字符串

using namespace std;

int main(int argc, char** argv) {
    // ============================================================
    // 1. 命令行参数检查
    // ============================================================
    // argc: 参数个数
    // argv: 参数列表
    // 例如：./simple_download http://example.com/file.txt output.txt
    // argc = 3
    // argv[0] = "./simple_download"
    // argv[1] = "http://example.com/file.txt"
    // argv[2] = "output.txt"
    
    if (argc < 2) {
        // 如果没有提供 URL，显示帮助信息
        cout << "用法: " << argv[0] << " <URL> [输出文件]" << endl;
        cout << "示例: " << argv[0] << " http://example.com/file.txt output.txt" << endl;
        cout << "      " << argv[0] << " minio://192.168.1.100:9000/bucket/file.png image.png" << endl;
        return 1;  // 返回 1 表示错误
    }

    // ============================================================
    // 2. 解析参数
    // ============================================================
    string url = argv[1];  // 第一个参数是 URL
    string output_file = (argc >= 3) ? argv[2] : "downloaded_file";  // 第二个参数是输出文件

    // ============================================================
    // 3. 验证 URL
    // ============================================================
    if (!myblob::cloud::Provider::isRemoteFile(url)) {
        cerr << "错误: 无效的URL格式" << endl;
        return 1;
    }

    // ============================================================
    // 4. 显示信息
    // ============================================================
    cout << "下载文件: " << url << endl;
    
    // ============================================================
    // 5. 创建云存储提供者
    // ============================================================
    // makeProvider 会根据 URL 自动识别云服务商类型
    // 并创建对应的提供者实例
    auto provider = myblob::cloud::Provider::makeProvider(url);
    
    // 检查是否创建成功
    if (!provider) {
        cerr << "错误: 无法创建provider" << endl;
        return 1;
    }

    // ============================================================
    // 6. 提取文件路径
    // ============================================================
    // URL 格式：http://example.com:8080/bucket/file.txt
    // 我们需要提取 /bucket/file.txt 部分
    
    string file_path = "/";
    if (url.find("://") != string::npos) {
        size_t slash_pos = url.find("://") + 3;  // 找到 "://" 后面的位置
        size_t path_pos = url.find("/", slash_pos);  // 找到路径开始的位置
        if (path_pos != string::npos) {
            file_path = url.substr(path_pos);  // 提取路径部分
        }
    }

    // ============================================================
    // 7. 执行下载
    // ============================================================
    auto response = provider->download(file_path);

    // ============================================================
    // 8. 检查下载结果
    // ============================================================
    if (!response.success) {
        cerr << "下载失败: " << response.error_message << endl;
        return 1;
    }

    // ============================================================
    // 9. 保存文件
    // ============================================================
    // 以二进制模式打开文件
    ofstream out_file(output_file, ios::binary);
    
    // 检查文件是否打开成功
    if (!out_file) {
        cerr << "错误: 无法创建输出文件 " << output_file << endl;
        return 1;
    }

    // 写入数据
    // cdata() 返回 const 指针（只读）
    // reinterpret_cast<const char*> 将 uint8_t* 转换为 char*
    out_file.write(
        reinterpret_cast<const char*>(response.body.cdata()),  // 数据指针
        response.body.size()  // 数据大小
    );
    
    // 关闭文件
    out_file.close();

    // ============================================================
    // 10. 显示结果
    // ============================================================
    cout << "下载成功!" << endl;
    cout << "文件大小: " << response.body.size() << " 字节" << endl;
    cout << "保存到: " << output_file << endl;

    return 0;  // 成功退出
}
```
