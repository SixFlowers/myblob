## 二、详细错误及修复

### 错误 1: defer.hpp 头文件错误

**错误信息**:
```
fatal error: function: 没有那个文件或目录
#include <function>
```

**原因**: `<function>` 不是标准库头文件，应该是 `<functional>`

**修复**: 
```cpp
// 修改前
#include <function>

// 修改后
#include <functional>
```

---

### 错误 2: DataVector 缺少拷贝功能

**错误信息**:
```
error: use of deleted function 'std::unique_ptr<_Tp [], _Dp>::operator='
```

**原因**: `DataVector` 使用 `unique_ptr` 管理内存，但缺少拷贝构造函数和赋值运算符

**修复**: 添加完整拷贝/移动语义
```cpp
DataVector(const DataVector& other) : _capacity(0), _size(0), _data(nullptr) {
    if (other._size > 0) {
        resize(other._size);
        std::memcpy(_data, other._data, other._size * sizeof(T));
    }
}

DataVector& operator=(const DataVector& other) {
    if (this != &other) {
        _dataOwned.reset();
        _data = nullptr;
        _capacity = 0;
        _size = 0;
        if (other._size > 0) {
            resize(other._size);
            std::memcpy(_data, other._data, other._size * sizeof(T));
        }
    }
    return *this;
}
```

### 错误 3: connection.hpp 缺少成员变量

**错误信息**:
```
error: 'use_tls' was not declared in this scope
return use_tls;
```

**原因**: 头文件中声明了 `usesTLS()` 方法，但类中缺少 `use_tls_` 成员变量

**修复**: 添加 `use_tls_` 到私有成员
```cpp
private:
    std::string host_;
    uint16_t port_;
    bool use_tls_;  // 新增
    int sockfd_;
    // ...
```

### 错误 4: http_client.hpp 缺少 HttpResponse 引用

**错误信息**:
```
error: 'HttpResponse' does not name a type
```

**原因**: `HttpClient` 使用 `HttpResponse` 但没有 include 对应的头文件

**修复**: 在 `http_client.hpp` 中添加
```cpp
#include "http_response.hpp"
#include "http_request.hpp"
```

### 错误 5: 函数名与系统调用冲突

**错误信息**:
```
error: cannot convert 'int' to 'std::shared_ptr<Connection>'
```

**原因**: `HttpClient::send()` 方法与系统调用 `send()` 同名，导致递归调用

**修复**: 
1. 重命名方法 `send` → `sendRequest`
2. 系统调用使用 `::send()` 明确调用

```cpp
// 头文件中
HttpResponse sendRequest(std::shared_ptr<Connection> conn, ...);

// 实现中
::send(sockfd, request_str.c_str(), request_str.size(), 0);
```

### 错误 8: parseRemoteInfo 没有解析主机地址

**错误信息**:
```
[ERROR] DNS lookup failed for: 
```

**原因**: `parseRemoteInfo` 函数只设置协议类型和端口，没有解析主机名

**修复**: 完整实现 URL 解析
```cpp
inline RemoteInfo parseRemoteInfo(const std::string& url) {
    RemoteInfo info;
    
    // ... 解析协议 ...
    
    // 新增：解析主机地址
    if (prefix_len > 0) {
        std::string remaining = url.substr(prefix_len);
        size_t slash_pos = remaining.find('/');
        size_t colon_pos = remaining.find(':');
        
        std::string host_port;
        // ... 提取 host:port ...
        
        size_t colon = host_port.find(':');
        if (colon != std::string::npos) {
            info.endpoint = host_port.substr(0, colon);
            info.port = static_cast<uint16_t>(std::stoi(port_str));
        } else {
            info.endpoint = host_port;
        }
    }
    
    return info;
}
```

### 错误 9: HTTPProvider 缺少 ConnectionManager

**错误信息**:
```
[ERROR] ConnectionManager or HttpClient not initialized
```

**原因**: `HTTPProvider` 构造函数没有正确传递 ConnectionManager 给父类

**修复**: 在 `makeProvider` 中创建静态 ConnectionManager
```cpp
std::unique_ptr<Provider> Provider::makeProvider(...) {
    static network::ConnectionManager conn_mgr;
    static network::HttpClient http_client;
    
    // ...
    return std::make_unique<HTTPProvider>(
        info.endpoint, info.port, info.https, conn_mgr, http_client);
}
```
